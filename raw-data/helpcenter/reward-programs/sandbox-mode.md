### [hashtag](#what-is-this-feature) What is this feature?

Sandbox Mode is a testing environment allowing you to test your loyalty program without awarding real customer points. When enabled, activities like sign-ups and orders will not generate points for regular customer accounts but only for designated test accounts.

### [hashtag](#why-use-this-feature) Why use this feature

Setting up a loyalty program involves careful planning and testing to ensure it creates the best customer experience. Sandbox Mode helps you:

- check

  Set up and test your loyalty program before launching it to all customers
- check

  Make changes to your existing program without affecting regular customers
- check

  Verify that point calculations and rewards work as expected
- check

  Avoid having to reset customer points and activities after testing

### [hashtag](#setting-up-and-using-sandbox-mode) Setting up and using Sandbox Mode

Before you begin, ensure you know which test accounts you want to use for your testing and have a clear plan for what aspects of your loyalty program you want to test.

1

**Enable Joy embed in the Theme editor**

- Access to the **Theme editor**
- Navigate to **App Embeds** to find the Joy embed named "**Joy Loyalty - Widget**" and turn it on
- Remember to save before switching back to Joy admin

2

**Enable Sandbox Mode**

- Locate the **Sandbox Mode** toggle switch in the **Reward programs** menu
- Click the toggle to turn Sandbox Mode on
- Review the confirmation message explaining what will happen
- Click "**Confirm**" to enable Sandbox Mode

When Sandbox Mode is enabled, you'll see a notification banner at the top of your Joy Loyalty dashboard reminding you that you're in testing mode.

3

**Add test accounts**

To effectively test your loyalty program, you'll need to designate specific accounts as test accounts:

1. Navigate to the Test Accounts section within Sandbox Mode
2. Click "Add Test Account"
3. Enter the account information for your test customer
4. Click "Save" to add the account to your test list
5. Repeat for additional test accounts as needed

### [hashtag](#sandbox-mode-use-case) Sandbox mode use case

Let's say you want to test a new rule that awards double points on a customer's first purchase:

1. Enable Sandbox Mode
2. Add your test account to the test accounts list
3. Create a new order using your test account
4. Check that the account receives double points for the first purchase
5. Create another order and verify that regular points are awarded
6. Make any adjustments to the rule as needed
7. When satisfied with the results, disable Sandbox Mode

### [hashtag](#faqs) FAQs

**Q: Will enabling Sandbox Mode affect existing customer points?**
A: No. Enabling Sandbox Mode only affects how new points are awarded. Existing points in customer accounts will remain unchanged.

**Q: Can I still see reporting data while in Sandbox Mode?**
A: Yes. All reporting features continue to function in Sandbox Mode; however, activities performed during this time will be tagged as "test" activities in reports.

**Q: How long can I keep Sandbox Mode enabled?**
A: You can keep Sandbox Mode enabled as long as needed for testing. Please note that regular customers won't earn points during this period.

**Q: Can I add or remove test accounts while Sandbox Mode is active?**
A: Yes. You can manage your test accounts at any time while Sandbox Mode is enabled.

**Q: If I disable Sandbox Mode, will test accounts continue to receive points?**
A: No. When Sandbox Mode is disabled, all accounts (including test accounts) will follow the standard loyalty program rules.

**Q: Should I hide the loyalty widget from customers during testing?**
A: Yes, if you don't want customers to see the loyalty widget in your store during testing, you should hide the widget by going to **Branding settings** and turning off the widget display for both desktop and mobile.
